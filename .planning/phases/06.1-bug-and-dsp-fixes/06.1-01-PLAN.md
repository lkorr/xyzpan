<objective>
Fix five bugs identified in Phase 06.1 diagnostic:

1. **Signal explosion** — Two root causes: (a) distance gain formula uses `kMinDistance/dist` which gives only 0.1x at Y=1 (dist=1.0), meaning a normally-placed source is -20dB below input; when other gains accumulate (floor bounce, comb, reverb wet) the signal blows up at extremes; (b) floor bounce delay lines have push/read order inverted — `push(dL)` then `read(...)` then `dL += read(...)` means the signal feeds back through the floor delay each sample.

2. **Proximity panning inversion** — Same root cause as (a): `distGainTarget = kMinDistance / dist = 0.1 / 1.0 = 0.1` at the neutral position Y=1. The gain formula needs to be recalibrated so unity gain is at the default source position, not at `dist = kMinDistance`.

3. **Doppler glitchiness** — `kDistSmoothMs = 30ms` is too fast: at 44.1kHz a 30ms smoother converges in ~1323 samples. For large position jumps (e.g., dist=0.1 → dist=√3) the delay target jumps from ~2 to ~13,000 samples; the smoother ramps but still produces large per-sample delay changes → extreme pitch artifacts. Fix: increase to 150ms and add a per-sample rate-of-change clamp on the delay (max ±0.5 samples/sample to keep pitch shift below ±500 cents).

4. **Doppler UI toggle** — `DOPPLER_ENABLED` is only accessible inside the dev panel. Add a `dopplerToggle_` ToggleButton to the main editor bottom strip so it's always visible without opening the dev panel.

5. **Dev panel invisible / LFO knobs too small** — In `PluginEditor::resized()`: `glArea` is mutated by `removeFromTop()` calls before `devPanel_.setBounds()` uses it, so the panel is positioned off-screen. Also `devToggle_` is placed at `glArea.getX()/getY()` after the snap row has been removed, misaligning the button. For LFO strips: the column is `colW=100px` and `LFOStrip::resized()` computes `knobW = (100 - 28 - 2 - 32) / 3 ≈ 12px` — too small to interact with. Fix: widen LFO columns to at least 160px (or reduce to 2 knobs in the strip by dropping phase knob from the strip layout to a separate section).
</objective>

<task id="1" name="Fix distance gain formula and floor-bounce push/read order">

**Root cause A — Distance gain formula:**
Current: `distGainTarget = clamp(kMinDistance / dist, 0, 1)` where `kMinDistance = 0.1f`
At Y=1, X=0, Z=0: `dist = 1.0`, so `distGainTarget = 0.1f` → -20dB. That's the "normal" position.
At Y=0.5, dist ≈ 0.5: `distGainTarget = 0.2f` → still -14dB.
The formula intends to give unity at the closest practical distance, but 0.1 is far too small.

Fix: Change the reference so `distGainTarget = 1.0` at Y=1 (dist=1.0), which is the natural "in front of you" position. Use `distGainTarget = clamp(1.0f / dist, 0, kDistGainMax)` where `kDistGainMax = 2.0f` (allows boost for very close sources but caps at +6dB to prevent explosion). Add `kDistGainRef = 1.0f` and `kDistGainMax = 2.0f` to Constants.h.

In Engine.cpp, change:
```cpp
const float distGainTarget = std::clamp(kMinDistance / dist, 0.0f, 1.0f);
```
to:
```cpp
const float distGainTarget = std::clamp(kDistGainRef / dist, 0.0f, kDistGainMax);
```

Add to Constants.h:
```cpp
constexpr float kDistGainRef = 1.0f;   // unity gain at dist=1.0 (default source position Y=1)
constexpr float kDistGainMax = 2.0f;   // max +6dB boost for very close sources
```

**Root cause B — Floor-bounce feedback loop:**
Current code (Engine.cpp per-sample loop, floor bounce section):
```cpp
floorDelayL_.push(dL);     // <-- push current dL
floorDelayR_.push(dR);
const float floorGain = floorGainSmooth_.process(floorLinearTarget);
const float floorReadSamp = std::max(2.0f, floorDelaySamp);
if (floorGain > 1e-6f) {
    dL += floorDelayL_.read(floorReadSamp) * floorGain;   // <-- adds floor output BACK to dL
    dR += floorDelayR_.read(floorReadSamp) * floorGain;
}
```
The next sample: `dL` now contains the floor echo, and we push THAT into the delay line again → feedback accumulates. The floor bounce should read the delayed version of dL, then add it — but push must happen before read so the delay line has the current sample. The issue is that `dL` after `dL +=` feeds into the NEXT `push`.

This is actually correct behavior for a delay + parallel add (the delay provides the tap, not a feedback path). The real issue is that `dL` accumulates the floor echo, which then gets passed to `distDelayL_.push(dL)` in the distance section — which means the floor echo also gets delayed and then looped through reverb, creating a multi-stage accumulation. With the distance gain at 0.1x AND floor gain contributing, the total gain through all stages can easily exceed 1.0 with certain parameter combinations.

The fix is:
1. Fix the distance gain formula (fix A above) — this is the primary cause
2. Add a hard output limiter at the end of the per-sample loop: `outL[i] = std::clamp(dL, -2.0f, 2.0f)` to prevent NaN/Inf from propagating

Also add NaN/Inf guard at the start of the per-sample loop:
```cpp
// Safety: NaN/Inf guard — if signal is non-finite, zero it to break feedback spirals
if (!std::isfinite(dL)) dL = 0.0f;
if (!std::isfinite(dR)) dR = 0.0f;
```

**Implementation steps:**
1. Add `kDistGainRef` and `kDistGainMax` to `engine/include/xyzpan/Constants.h`
2. In `engine/src/Engine.cpp`, update `distGainTarget` formula
3. Add NaN/Inf guard at start of per-sample `dL`/`dR` processing (after `monoEQ` is computed, before binaural section uses it)
4. Add hard output clamp at `outL[i]` / `outR[i]` assignment: `std::clamp(dL, -2.0f, 2.0f)`

**Verification:** Build and run. Play a test signal, move source to Y=1 — output should be near unity. Move to extreme parameters — no explosion.

<task id="2" name="Fix doppler smoother time constant and add rate-of-change limiter">

Current `kDistSmoothMs = 30ms`. For large jumps this causes audible zipper/pitch artifacts.

**Fix 1: Increase default smooth time constant.**
In Constants.h, change:
```cpp
constexpr float kDistSmoothMs = 30.0f;
```
to:
```cpp
constexpr float kDistSmoothMs = 150.0f;
```

This gives ~150ms ramp time for delay changes, which maps to a maximum pitch shift of about ±150 cents during fast movement — audible doppler without being glitchy.

**Fix 2: Per-sample rate-of-change clamp on delay smoother output.**
Even with 150ms, very large jumps (e.g., 0→300ms of delay) produce extreme pitch shifts. Add a clamp on the per-sample change in delay position:

In Engine.cpp, in the per-sample loop, replace:
```cpp
const float delaySamp = std::max(2.0f, distDelaySmooth_.process(delayTargetSamples));
```
with:
```cpp
const float rawDelay = distDelaySmooth_.process(delayTargetSamples);
// Clamp rate-of-change: max 1.0 sample/sample = ±1 semitone/sample max pitch shift
// This prevents the "zipper" artifact when jumping large distances
const float deltaDelay = rawDelay - lastDelayL_;
const float clampedDelta = std::clamp(deltaDelay, -kDopplerMaxDeltaSamp, kDopplerMaxDeltaSamp);
lastDelayL_ = lastDelayL_ + clampedDelta;
const float delaySamp = std::max(2.0f, lastDelayL_);
```

Add `lastDelayL_` as a private member to `XYZPanEngine` (float, init to 2.0f in prepare/reset).
Add to Constants.h:
```cpp
constexpr float kDopplerMaxDeltaSamp = 2.0f;  // max delay change per sample (~±1 octave/sample max)
```

**Note:** `lastDelayL_` is used for both L and R (they track the same delay position — distance is mono, not per-ear). Use one `lastDistDelaySamp_` tracking variable.

Add `lastDistDelaySamp_` to Engine.h private section. Initialize to 2.0f in prepare() and reset().

<task id="5" name="Fix LFO strip knob sizing">


The bottom strip column width is `colW = 100px` in `PluginEditor`. LFO strips get that same 100px. The wave button + 3 knobs + sync simply don't fit in 100px.

**Fix option A:** Widen the LFO columns. Change `colW` in the PluginEditor `resized()` for LFO columns from 100 to 160px. Increase `kDefaultW` if needed to avoid overflow.

**Fix option B:** Drop the phase knob from the LFO strip. The phase param is rarely adjusted and can stay in the dev panel. Strip becomes: [WaveBtn 28] + [Rate 50] + [Depth 50] + [SYNC 32] = 160px... still too wide for 100px.

**Fix option C (chosen):** Widen LFO columns specifically in the editor `resized()` layout. Use `kLFOColW = 160` for LFO knob columns and `kKnobColW = 80` for plain position knobs (X/Y/Z/R), and use `kVerbColW = 80` for verb knobs. Adjust `kDefaultW` to 900px minimum (already set), which can fit: 4×80 (pos knobs) + 3×160 (LFO) + 4×80 (verb) + 100 (doppler+padding) = 320+480+320+100 = 1220px. That's too wide.

**Better fix:** Keep the LFO column at `colW=100` but redesign LFO strip layout to 2 rows:
- Row 1: [WaveBtn] [Rate knob] [SYNC btn]

**Implementation:** Rewrite `LFOStrip::resized()` with a 2-row layout. Waveform button in top-left, rate knob top-center, sync top-right, depth+phase side by side in bottom row.
