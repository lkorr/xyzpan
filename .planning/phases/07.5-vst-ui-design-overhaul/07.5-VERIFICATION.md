## Human Verification Required

### 1. Window opens at landscape proportions

**Test:** Build and launch standalone: `build/plugin/XYZPan_artefacts/Release/Standalone/XYZPan.exe`
**Expected:** Window is noticeably wider than tall, approximately 1100x750. Fits on a 1080p monitor without being cropped.
**Why human:** `kDefaultH=750` and `setSize(kDefaultW, kDefaultH)` are confirmed in code, but actual window placement and OS DPI scaling require visual verification.

### 2. ToggleButton Alchemy theme (Always Face Observer + Sync + Snap buttons)

**Test:** In standalone, click the "Always Face Observer" toggle button in the orbit controls section.
**Expected:** When ON: gold/warm-gold border with bright gold text. When OFF: muted purple/bronze border with parchment text. No checkbox, tick mark, or checkmark graphic visible — pure text button with colored border.
**Why human:** drawToggleButton implementation replaces the JUCE V4 tick-box default; correct rendering must be visually confirmed.

### 3. ComboBox preset dropdown appearance

**Test:** Click the preset ComboBox at the top of the window.
**Expected:** Dark iron/dark-plum background, muted-purple (bronze) rounded border, warm gold downward-pointing triangle arrow in the right side. Visual matches the Alchemy palette.
**Why human:** drawComboBox code is correct but actual pixel rendering requires display.

### 4. PopupMenu item theming

**Test:** Open the preset dropdown so the popup list appears.
**Expected:** Items on dark background, lavender-grey (parchment) text, bronze fill highlight on hovered items, bright gold text on highlighted item, small gold dot at left margin for the currently selected/ticked item. Separator is a thin bronze line.
**Why human:** drawPopupMenuItem implementation is verified; popup overlay rendering requires visual inspection.

### 5. Snap button active state feedback

**Test:** Click "XY" snap button, then "XZ", then "YZ" in the top-right corner of the OpenGL view.
**Expected:** The clicked button shows gold border (active). The other two show bronze/muted-purple border. Clicking a new button moves the gold border to it.
**Why human:** updateSnapButtonStates() wiring confirmed in code; visual toggle state rendering requires display.

### 6. Orbit controls show rotary knobs

**Test:** In the bottom-left "STEREO ORBIT" panel, inspect the Width, Offset, and Phase controls.
**Expected:** Three circular rotary knobs (like the position X/Y/Z knobs and reverb knobs). The Speed control below the LFO strips remains a horizontal slider.
**Why human:** RotaryHorizontalVerticalDrag is set in configOrbitKnob lambda; rendered widget type needs visual confirmation.

### 7. Multi-instance LookAndFeel safety

**Test:** Load two instances of XYZPan in a DAW (e.g., two plugin tracks in Reaper).
**Expected:** Both instances display correct Alchemy dark/gold theme. Closing or resizing one instance does not corrupt the other's appearance.
**Why human:** setDefaultLookAndFeel removal confirmed by code grep, but inter-instance isolation requires runtime test with multiple loaded instances.
