# Phase 07.5 Plan 01: Window Layout and Safety Fixes Summary

**Plugin window resized 1100x1100->1100x750 with Layout struct unifying paint/resized geometry and multi-instance-safe LookAndFeel**

## Accomplishments

- Plugin default window is now 1100x750 (landscape) — usable on standard 1080p displays
- Removed `setDefaultLookAndFeel(&lookAndFeel_)` from ctor and `setDefaultLookAndFeel(nullptr)` from dtor — multiple plugin instances no longer corrupt each other's LookAndFeel
- Extracted `Layout::compute(totalW, totalH)` struct with all structural geometry (contentY, leftColH, posSectionH, miscTop, bottomY, devW, reverbX, orbitTotalW, contentTop) — both `paint()` and `resized()` call it, eliminating the duplication hazard from Pitfall 5
- LFO strip heights verified adequate at 750px: position section LFO strips ~192px (>109px min), orbit LFO strips ~184px (>109px min)

## Task Commits

1. **Task 1: Window resize, setDefaultLookAndFeel removal, and Layout struct extraction** - `e681ab7` (feat)

**Plan metadata:** (docs commit follows)

## Files Created/Modified

- `plugin/PluginEditor.h` - kDefaultH changed to 750; Layout struct added with compute() static method
- `plugin/PluginEditor.cpp` - setDefaultLookAndFeel removed from ctor+dtor; Layout::compute() implementation added; paint() and resized() refactored to use lo = Layout::compute()

## Decisions Made

- `kDefaultH` changed from 1100 to 1100 to 750. The resizeLimits already allowed 750 as minimum height so no constraint changes were needed.
- `setDefaultLookAndFeel` was the only change needed for multi-instance safety. JUCE propagates LookAndFeel to children automatically through `addAndMakeVisible` — no explicit per-child setLookAndFeel calls needed.
- `Layout::compute` takes explicit `totalW, totalH` parameters (not calling getWidth/getHeight internally) — keeps it a pure value computation. Call sites pass `getWidth(), getHeight()`.
- Removed unused `leftColH` local variable in `resized()` that was flagged as unreferenced by MSVC (C4189) after migrating to `lo.miscTop`.
- `posSectionBottom` in resized() simplified: original `leftColTop + kSectionHdrH + (posSectionH - kSectionHdrH) - lfoSpeedRowH` reduces to `miscTop - lfoSpeedRowH`. Applied the simplification.
