<task type="auto">
  <name>Task 1: Per-block coefficient pre-computation refactor</name>
  <files>engine/src/Engine.cpp, engine/CMakeLists.txt</files>
  <action>
Refactor Engine::process() to move ALL setCoefficients() calls from the per-sample loop to the per-block preamble. This is the single highest-impact optimization. The changes are:

**A. Mono path (lines ~938-1024 in current code) — move to per-block:**

In the per-block preamble (before `for (int i = 0; i < numSamples; ++i)`), pre-compute all position-derived EQ targets using block-start values. These values are already computed per-sample in the loop but are constant or near-constant within a block because:
- Parameters don't change mid-block (setParams is called once per block)
- LFO modulation within a 64-128 sample block is negligible for coefficient purposes (smoothers handle transitions)

Move these per-block (BEFORE the sample loop):
1. `presenceShelf_.setCoefficients(...)` — uses presenceGainDb from position
2. `earCanalPeak_.setCoefficients(...)` — uses earCanalGainDb from position
3. `pinnaNotch_.setCoefficients(...)` — uses pinnaGainDb from position
4. `pinnaNotch2_.setCoefficients(...)` — uses pinnaN2Freq from position
5. `pinnaShelf_.setCoefficients(...)` — uses shelfGainDb from position
6. `nearFieldLF_L_.setCoefficients(...)` and `nearFieldLF_R_.setCoefficients(...)` — near-field ILD, 2 calls
7. `shadowL_.setCoefficients(...)` and `shadowR_.setCoefficients(...)` — head shadow SVFs
8. `rearSvfL_.setCoefficients(...)` and `rearSvfR_.setCoefficients(...)` — rear shadow SVFs
9. `airLPF_L_.setCoefficients(...)`, `airLPF_R_.setCoefficients(...)`, `airLPF2_L_.setCoefficients(...)`, `airLPF2_R_.setCoefficients(...)` — air absorption

For the mono path coefficients, compute block-start modulated position ONCE at the top of process() (after LFO setup but before the loop). Use the block-start `currentParams.x/y/z` to derive:
- `blkModX/Y/Z` = clamp(params.x/y/z + lfoX/Y/Z.peek() * depth, -1, 1) — but since LFO peek isn't available, use `currentParams.x/y/z` directly for coefficient computation. The LFO delta within a single block is negligible for EQ coefficients.
- From these: horizontalMag, rearFactor, presenceGainDb, earCanalGainDb, pinnaGainDb, shelfGainDb, pinnaN1Freq, modDist, modDistFrac, modProximity, effectiveAzimuth, shadowCutoffTarget, ildTarget, rearCutoffTarget, airCutoff, etc.
- Then call all setCoefficients on the flat (L-pipeline) filters.

For the mono path, the per-sample loop should ONLY call `.process()` on filters — never `.setCoefficients()`.

The ILD target and ITD target still need per-sample smoothing (itdSmooth_.process, ildGainSmooth_.process, etc.) because the smoothers must tick every sample. But the TARGET values passed to the smoothers can be computed once per block. The smoothers' `.process(target)` calls remain per-sample — this is correct and cheap.

However, the smoothed outputs (shadowCutoff, rearCutoff) currently drive per-sample SVF setCoefficients. After the refactor, the SVF coefficients are set once per block using the block-start target values. The SVFs' `.process()` still runs per-sample. This means the SVF cutoff changes in steps (once per block) instead of smoothly. This is acceptable because:
- At 64 samples/block = 1.45ms, the step is below the ~5ms smoothing constant
- The smoother was smoothing a slowly-changing value anyway

**IMPORTANT: Keep the per-sample smoother .process() calls** — they must tick every sample to maintain correct state. Just don't feed their output into setCoefficients per-sample. Instead, set coefficients per-block from the block-start target, and let the smoothers tick (their state is used for other purposes like ILD gain multiplication and ITD delay reading).

Wait — on closer inspection, the ITD smoother output directly drives delay read positions (per-sample) and the ILD gain smoother output directly multiplies samples (per-sample). These are NOT coefficient updates — they're cheap multiplies and delay reads. Keep them per-sample. Only the setCoefficients calls on SVF/biquad/OnePoleLP need to move to per-block.

**B. Stereo path — processBinauralForSource() refactor:**

The function currently calls setCoefficients on 6 biquads + 2 near-field biquads + 2 head shadow SVFs + 2 rear shadow SVFs per call. It's called TWICE per sample in stereo mode (once for L pipeline, once for R pipeline).

Refactor: Move coefficient computation OUT of processBinauralForSource(). In the per-block preamble when stereoActive, compute L-node and R-node positions from block-start params, then:
1. Call setCoefficients on L pipeline (presenceShelf_, earCanalPeak_, etc.) with L-node-derived values
2. Call setCoefficients on R pipeline (srcR_.presenceShelf, srcR_.earCanalPeak, etc.) with R-node-derived values

Then in processBinauralForSource(), remove ALL setCoefficients calls. The function only calls .process() on pre-configured filters.

**C. processDistanceForNode() refactor:**

Currently calls airLPF setCoefficients per-sample (4x OnePoleLP.setCoefficients containing std::exp). Move to per-block:
1. For mono path: compute air cutoff from block-start distFrac, call setCoefficients on airLPF_L/R/2_L/2_R once
2. For stereo path: compute air cutoff for each node from block-start node positions, call setCoefficients on both pipeline's air LPFs

Remove setCoefficients from processDistanceForNode(). The function only calls .process().

**D. dB-to-linear conversions:**

Move these out of the per-sample loop to per-block:
1. `std::pow(10.0f, currentParams.chestGainDb / 20.0f)` — chest bounce
2. `std::pow(10.0f, currentParams.floorGainDb / 20.0f)` — floor bounce
3. `std::pow(10.0f, -currentParams.ildMaxDb / 20.0f)` — ILD (used in both mono path and processBinauralForSource)
4. `std::pow(10.0f, currentParams.auxSendGainMaxDb / 20.0f)` — aux reverb send

Pre-compute as `const float chestGainLin`, `floorGainLin`, `ildGainBase`, `auxMaxBoostLin` in the per-block preamble.

**E. CMake compiler flags:**

Add to engine/CMakeLists.txt after `target_compile_features(xyzpan_engine PUBLIC cxx_std_20)`:

```cmake
# Release optimization: full optimization, AVX2 SIMD, relaxed float for auto-vectorization
if(MSVC)
    target_compile_options(xyzpan_engine PRIVATE
        $<$<CONFIG:Release>:/O2 /arch:AVX2 /fp:fast>
    )
endif()
```

**F. Correctness invariants to maintain:**

1. `const float sr = static_cast<float>(sampleRate)` — always use the member sampleRate, never a captured value. This ensures INFRA-03 (prepare() with new sr takes effect on next process()).
2. `std::isfinite()` guards on dL/dR must remain — they still work correctly under /fp:fast on MSVC.
3. OnePoleSmooth `.process()` calls MUST remain per-sample — they tick state for gain/delay smoothing.
4. LFO `.tick()` calls MUST remain per-sample — they advance phase accumulators.
5. FeedbackCombFilter `.process()`, FractionalDelayLine `.push()/.read()`, FDNReverb `.processSample()` all remain per-sample — these are the actual audio processing.
6. Zero new heap allocations in process().
