**Eliminated per-sample transcendental math (cos/sin/pow/sqrt/tan/exp) from the audio loop by moving all filter coefficient updates to a per-block preamble, plus AVX2/O2/fp:fast Release flags and 3 INFRA-03 sample-rate regression tests.**

## Accomplishments
- Moved all BiquadFilter, SVFLowPass, and OnePoleLP setCoefficients() calls from the per-sample loop to a per-block preamble in Engine::process()
- Pre-computed 4 dB-to-linear std::pow conversions once per block (chest, floor, ILD, aux)

## Decisions Made
- Block-start unmodulated position is used for per-block coefficients. LFO modulates within the per-sample loop, but the delta over one block is negligible for EQ coefficients.
- ildGainBase_ is a new Engine member (not local to process()) so that processBinauralForSource() can access the cached value without a parameter change.
- The stereo coefficient block runs unconditionally every block (not guarded by stereoActive) because stereoWidth can transition, and when it is 0 the L-node position equals the object center so values are identical.
- /fp:fast is safe here: the std::isfinite() guards on delay lines still work correctly under MSVC /fp:fast per the compiler documentation.
