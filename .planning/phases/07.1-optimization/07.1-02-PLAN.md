Move the stereo orbit angular smoother computation from per-sample to per-block. Currently in the per-sample loop (when stereoActive), the following transcendentals execute every sample:

```cpp
// Per sample currently:
phaseSmCos_ = std::cos(phaseRad) * b + phaseSmCos_ * angularSmA_;   // cos
phaseSmSin_ = std::sin(phaseRad) * b + phaseSmSin_ * angularSmA_;   // sin
offsetSmCos_ = std::cos(offsetRad) * b + offsetSmCos_ * angularSmA_; // cos
offsetSmSin_ = std::sin(offsetRad) * b + offsetSmSin_ * angularSmA_; // sin
const float smoothedPhase  = std::atan2(phaseSmSin_, phaseSmCos_);    // atan2

```

These are 6-10 transcendental calls per sample. `currentParams.stereoOrbitPhase` and `stereoOrbitOffset` are parameter values that don't change mid-block, so the angular smoother needs only one update per block.

**Refactor:**

Move to the per-block preamble (after `stereoWidthSmooth_` setup, before the sample loop):

```cpp
// Per-block orbit angular smoother update
float smoothedPhase = 0.0f, smoothedOffset = 0.0f;
float rPhaseOffset = 0.0f;
if (smoothedWidth > 0.001f && inputR != nullptr) {
    const float b = 1.0f - angularSmA_;
    {
        const float phaseRad = currentParams.stereoOrbitPhase * kTwoPI;
        phaseSmCos_ = std::cos(phaseRad) * b + phaseSmCos_ * angularSmA_;
        phaseSmSin_ = std::sin(phaseRad) * b + phaseSmSin_ * angularSmA_;
    }
    {
        const float offsetRad = currentParams.stereoOrbitOffset * kTwoPI;
        offsetSmCos_ = std::cos(offsetRad) * b + offsetSmCos_ * angularSmA_;
        offsetSmSin_ = std::sin(offsetRad) * b + offsetSmSin_ * angularSmA_;
    }
    smoothedPhase  = std::atan2(phaseSmSin_, phaseSmCos_);
    smoothedOffset = std::atan2(offsetSmSin_, offsetSmCos_);
    rPhaseOffset = kPI + smoothedPhase;
}
```

Note: The angular smoother IIR (`cos*b + state*a`) is a first-order filter. Advancing it once per block (every 1.45ms at 64 samples/44.1kHz) vs per-sample is valid because the 5ms smoothing time constant means the filter response time is longer than the block. The per-block update gives the same steady-state convergence, just with slightly coarser steps that are inaudible.

In the per-sample loop stereo branch, use the pre-computed `smoothedPhase`, `smoothedOffset`, `rPhaseOffset` variables directly — remove the cos/sin/atan2 calls from the loop body.

For the XZ/YZ orbit rotation cos/sin calls: if `orbitDepXZ` or `orbitDepYZ` are constant within the block (they are — they come from a smoother whose target is a per-block parameter), pre-compute `cosXZ/sinXZ` and `cosYZ/sinYZ` per-block too. The orbit LFO raw values (orbitRawXZ, orbitRawYZ) change per-sample because the LFO ticks per-sample. BUT: the orbit angle = orbitRawXZ * orbitDepXZ * PI, and the LFO rate at typical values (0.01-50Hz) changes by at most 50/44100 = 0.001 per sample. At 128 samples per block, the total angle change is ~0.13 radians — the rotation matrix change is small enough that per-block is acceptable.

HOWEVER: if the orbit LFO tick is per-sample (which it must be for phase accuracy), we need the per-sample orbit angle. So for XZ/YZ rotation: keep the orbit LFO ticks per-sample, but move them to where they currently are (already per-sample). Only the PHASE/OFFSET angular smoothers move to per-block. The cos/sin of the orbit angle XZ/YZ remain per-sample because they depend on the per-sample LFO output.

Use `std::chrono::steady_clock` for timing. Pattern from research:

```cpp
#include <catch2/catch_test_macros.hpp>
#include "xyzpan/Engine.h"
#include "xyzpan/Types.h"
#include <chrono>
#include <vector>

using namespace xyzpan;

TEST_CASE("Engine throughput: 128 samples at 44.1kHz under 10% CPU", "[performance]") {
    XYZPanEngine engine;
    engine.prepare(44100.0, 128);

    std::vector<float> inBuf(128, 0.5f);
    std::vector<float> outL(128), outR(128);
    const float* inputs[1] = { inBuf.data() };

    EngineParams params;
    params.x = 0.3f; params.y = 0.8f; params.z = 0.1f;
    engine.setParams(params);

    // Warm up caches and branch predictors
    for (int i = 0; i < 100; ++i)
        engine.process(inputs, 1, outL.data(), outR.data(), nullptr, nullptr, 128);

    constexpr int kTrials = 1000;
    auto t0 = std::chrono::steady_clock::now();
    for (int i = 0; i < kTrials; ++i)
        engine.process(inputs, 1, outL.data(), outR.data(), nullptr, nullptr, 128);
    auto t1 = std::chrono::steady_clock::now();

    const double usPerBlock = std::chrono::duration<double, std::micro>(t1 - t0).count() / kTrials;
    // Budget: 128/44100 = 2902us. Target: <10% CPU = <290us
    INFO("us per 128-sample block: " << usPerBlock);
    CHECK(usPerBlock < 290.0);
}
```
