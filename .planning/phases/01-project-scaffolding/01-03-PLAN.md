  <name>Task 1: Implement Engine pass-through and PluginProcessor bridge</name>
  <files>engine/src/Engine.cpp, plugin/PluginProcessor.cpp</files>
  <action>
    Implement Engine.cpp with full Phase 1 pass-through behavior per 01-RESEARCH.md Pattern 5:

    prepare(double sampleRate, int maxBlockSize):
    - Store sampleRate and maxBlockSize
    - Resize monoBuffer to maxBlockSize samples (pre-allocate, NO allocation in process())

    setParams(const EngineParams& params):
    - Store params into currentParams

    process(const float* const* inputs, int numInputChannels, float* outL, float* outR, int numSamples):
    - If numInputChannels >= 2 and inputs[1] != nullptr: sum to mono into monoBuffer (0.5 * (left + right))
    - If numInputChannels == 1 or inputs[1] == nullptr: use inputs[0] directly as mono source
    - Copy mono source to both outL and outR (pass-through)
    - Call toSpherical(currentParams.x, currentParams.y, currentParams.z) -- validates the coordinate conversion is wired in, even though the result is unused in Phase 1. Store in a local variable (the compiler won't optimize it out with side effects from the function call). This proves SETUP-03: the engine has access to coordinate conversion.
    - Handle edge case: if inputs[0] is nullptr, zero both output buffers

    reset():
    - No-op in Phase 1 (no state to clear). Add a comment: "// Phase 2+: reset delay lines, filter states"

    Update plugin/PluginProcessor.cpp processBlock() to properly wire to engine:
    - Verify ScopedNoDenormals is first line
    - Snapshot X, Y, Z from atomic param pointers into EngineParams
    - Build input pointer array from buffer channels
    - Call engine.setParams(params)
    - Call engine.process(inputs, numInputChannels, outL, outR, numSamples)
    - Ensure prepareToPlay() calls engine.prepare(sampleRate, samplesPerBlock)
    - Ensure releaseResources() calls engine.reset()

    IMPORTANT: No heap allocation in process(). monoBuffer is pre-sized in prepare(). No std::vector resize, no new, no malloc in the process call.
