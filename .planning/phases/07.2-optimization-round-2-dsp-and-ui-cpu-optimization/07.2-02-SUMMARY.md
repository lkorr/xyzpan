**XYZPanGLView throttled to 30fps when idle / 60fps when moving with single sphereShader bind and shared uniform upload per frame (was per-draw)**

## Accomplishments
- Frame throttle cuts GL thread CPU by ~50% when plugin window is open but source is stationary
- Batched sphere/cone draws reduce uniform uploads from ~48 to ~24 per frame (shared projection/view/lightDir uploaded once, not per draw)
- Camera orbit drags properly detected (isDraggingCamera_) so they maintain 60fps smoothness
- Bridge read from throttle section reused for rendering (eliminates redundant atomic acquire per frame)

## Decisions Made
- Frame throttle uses early-return approach inside `setContinuousRepainting(true)` loop rather than dynamically toggling it — simpler and avoids GL thread race conditions
- `isDraggingCamera_` member tracks camera drag separately from source drag so orbit navigation stays at 60fps even when source position is unchanged
- `drawCone` temporarily binds vaoCone_ mid-batch then restores vaoSphere_ to maintain the invariant that vaoSphere_ is always bound between batch begin and end blocks
- 7 pre-existing test failures confirmed unchanged — all in TestDistanceProcessing and TestPhase3Integration, unrelated to GL rendering
