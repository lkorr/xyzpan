**2. Modify `ui/XYZPanGLView.cpp` -- frame rate throttle at top of renderOpenGL():**

Keep `setContinuousRepainting(true)` in the constructor (do NOT change to false -- this approach is safer per the research). The throttle is implemented INSIDE renderOpenGL() by early-returning when the minimum frame interval has not elapsed.

At the very top of `renderOpenGL()`, after the `jassert(juce::OpenGLHelpers::isContextActive());` line and BEFORE the viewport setup, add:
```cpp
// Frame rate throttle: 60fps when position is moving, 30fps when idle.
// Keeps setContinuousRepainting(true) for simplicity -- just skips frames.
{
    const auto currentSnap = bridge_.read();
    const bool positionChanged =
        currentSnap.x != lastSnap_.x ||
        currentSnap.y != lastSnap_.y ||
        currentSnap.z != lastSnap_.z ||
        currentSnap.stereoWidth != lastSnap_.stereoWidth ||
        currentSnap.lNodeX != lastSnap_.lNodeX ||
        currentSnap.lNodeY != lastSnap_.lNodeY ||
        currentSnap.rNodeX != lastSnap_.rNodeX ||
        currentSnap.rNodeY != lastSnap_.rNodeY;
    lastSnap_ = currentSnap;

    const double now = juce::Time::getMillisecondCounterHiRes();
    // 60fps = 16.67ms when active, 30fps = 33.33ms when idle
    const double minInterval = (positionChanged || isDraggingSource_) ? 16.0 : 33.0;
    if (now - lastRenderTime_ < minInterval) return;
    lastRenderTime_ = now;
}
```

NOTE: The `bridge_.read()` call in this throttle section means the existing `bridge_.read()` later in the function (line ~145) will get the same data. This is fine -- the bridge is a double-buffer and `read()` just reads the current write index. There is no "consume" semantic. However, to avoid a redundant atomic read, store the result and reuse it. Replace the later line:
```cpp
const auto snap = bridge_.read();
```
with:
```cpp
const auto& snap = lastSnap_;  // Already read in throttle section above
```

**3. Modify `ui/XYZPanGLView.cpp` -- batch sphere/cone draw calls:**

Refactor the draw helpers to support batched rendering. The approach:

**a) Modify `drawSphere()` to accept a `bool batchMode` parameter (default false for backwards compat). When batchMode is true, it skips shader->use(), projection/view/lightDir uniforms, and VAO bind/unbind -- only sets per-instance uniforms (model, nodeColor, opacity) and calls glDrawElements.**

Replace the `drawSphere()` implementation:
```cpp
void XYZPanGLView::drawSphere(const glm::vec3& position, float radius,
                               const glm::vec3& color, float opacity)
{
    if (!sphereShader_ || vaoSphere_ == 0 || sphereIndexCount_ == 0) return;

    const glm::mat4 model = glm::scale(
        glm::translate(glm::mat4(1.0f), position),
        glm::vec3(radius));

    sphereShader_->setUniformMat4("model", glm::value_ptr(model), 1, GL_FALSE);
    sphereShader_->setUniform("nodeColor", color.r, color.g, color.b);
    sphereShader_->setUniform("opacity", opacity);
    glDrawElements(GL_TRIANGLES, sphereIndexCount_, GL_UNSIGNED_INT, nullptr);
}
```

Replace the `drawSphereWithModel()` implementation:
```cpp
void XYZPanGLView::drawSphereWithModel(const glm::mat4& model,
```
