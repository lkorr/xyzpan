       - **Re-prepare smoothers if smoothing time params changed (per-block preamble):**
         This is critical for dev panel smoothing controls to be functional. The
         smoothers are initialized in prepare() with default constants, but the
         user can change smoothMs_ITD/Filter/Gain via APVTS at any time. Without
         this check, those controls would be dead.
         ```
         // Re-prepare smoothers when smoothing time parameters change.
         // OnePoleSmooth::prepare() only recomputes a_ and b_ coefficients
         // from the new time constant -- cheap (one exp() call per changed param).
         if (currentParams.smoothMs_ITD != lastSmoothMs_ITD_) {
             itdSmooth_.prepare(currentParams.smoothMs_ITD, static_cast<float>(sampleRate_));
             lastSmoothMs_ITD_ = currentParams.smoothMs_ITD;
         }
         if (currentParams.smoothMs_Filter != lastSmoothMs_Filter_) {
             shadowCutoffSmooth_.prepare(currentParams.smoothMs_Filter, static_cast<float>(sampleRate_));
             lastSmoothMs_Filter_ = currentParams.smoothMs_Filter;
         }
         if (currentParams.smoothMs_Gain != lastSmoothMs_Gain_) {
             ildGainSmooth_.prepare(currentParams.smoothMs_Gain, static_cast<float>(sampleRate_));
             lastSmoothMs_Gain_ = currentParams.smoothMs_Gain;
         }
         ```
         NOTE: Do NOT call reset() on the smoothers here -- only prepare(). Calling
         reset() would snap the smoother state to a new value and cause an audible
         click. prepare() updates the time constant while preserving the current
         smoothed state.

       - After smoother re-preparation, compute per-block targets:
         ```
         float x = currentParams.x;
         float y = currentParams.y;
         float dist = computeDistance(x, y, currentParams.z);

         // ITD target (only far ear delayed)
         float itdTargetSamples = currentParams.maxITD_ms
                                * std::sin(std::abs(x) * (PI / 2.0f))
                                * static_cast<float>(sampleRate) / 1000.0f;

         // Head shadow cutoff target (far ear only)
         float shadowCutoffTarget = kHeadShadowFullOpenHz
                                  + (currentParams.headShadowMinHz - kHeadShadowFullOpenHz)
                                  * std::abs(x);

         // ILD gain target (far ear only, proximity-dependent)
         float proximity = 1.0f - (dist - kMinDistance) / (kSqrt3 - kMinDistance);
         proximity = std::clamp(proximity, 0.0f, 1.0f);
         float ildMaxLinear = std::pow(10.0f, -currentParams.ildMaxDb / 20.0f);
         float ildGainTarget = 1.0f - (1.0f - ildMaxLinear) * std::abs(x) * proximity;

         // Rear shadow cutoff target (both ears, only when Y < 0)
         float rearAmount = std::max(0.0f, -y);
         float rearCutoffTarget = kRearShadowFullOpenHz
                                + (currentParams.rearShadowMinHz - kRearShadowFullOpenHz)
                                * rearAmount;
         ```

       - Per-sample loop:
         ```
         for (int i = 0; i < numSamples; ++i) {
             float mono = monoIn[i];

             // Smooth parameters
             float itdSamples = itdSmooth_.process(itdTargetSamples);
             float shadowCutoff = shadowCutoffSmooth_.process(shadowCutoffTarget);
             float ildGain = ildGainSmooth_.process(ildGainTarget);
             float rearCutoff = rearCutoffSmooth_.process(rearCutoffTarget);

             // Push mono into both delay lines
             delayL_.push(mono);
             delayR_.push(mono);

             // Read with ITD: far ear delayed, near ear at 0
             float dL, dR;
             if (x > 0.0f) {
                 // Source right: left ear is far
                 dL = delayL_.read(itdSamples);
                 dR = delayR_.read(0.0f);
             } else if (x < 0.0f) {
                 // Source left: right ear is far
                 dL = delayL_.read(0.0f);
                 dR = delayR_.read(itdSamples);
             } else {
                 dL = delayL_.read(0.0f);
                 dR = delayR_.read(0.0f);
             }

             // Apply ILD to far ear
             if (x > 0.0f) dL *= ildGain;
             else if (x < 0.0f) dR *= ildGain;

             // Update SVF coefficients (per-sample for smooth modulation)
             // Near ear stays wide open, far ear gets shadow cutoff
             if (x >= 0.0f) {
                 shadowL_.setCoefficients(shadowCutoff, sampleRate);
                 shadowR_.setCoefficients(kHeadShadowFullOpenHz, sampleRate);
             } else {
                 shadowL_.setCoefficients(kHeadShadowFullOpenHz, sampleRate);
                 shadowR_.setCoefficients(shadowCutoff, sampleRate);
             }
             dL = shadowL_.process(dL);
             dR = shadowR_.process(dR);

             // Rear shadow (both ears equally)
             rearSvfL_.setCoefficients(rearCutoff, sampleRate);
             rearSvfR_.setCoefficients(rearCutoff, sampleRate);
             dL = rearSvfL_.process(dL);
             dR = rearSvfR_.process(dR);

             outL[i] = dL;
             outR[i] = dR;
         }
         ```

If profiling shows this is too expensive, it can be optimized to per-N-samples in a later phase — but correctness first. The smoothers already limit the rate of change.

       IMPORTANT: For the x == 0 case and smooth transitions through zero, use the smoothed ITD value (which smoothly approaches 0) rather than branching hard on raw x. Consider using the smoothed shadow cutoff approaching kHeadShadowFullOpenHz. The branch on x sign determines WHICH ear is far/near, but the smoothed values handle the transition magnitude. If the executor finds a cleaner approach (e.g., computing separate L/R targets outside the branch), that is acceptable.
